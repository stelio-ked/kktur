Guia Prático: Resolvendo o Erro de Build no Easypanel

Passo a passo técnico para corrigir a falha de compilação da imagem Docker do GitHub

1 Qual é a Causa do Erro?

O log do seu deploy acusa:

ERROR: failed to build: failed to solve: failed to compute cache key: failed to calculate checksum of ref ... "/app/dist": not found

O seu arquivo Dockerfile tenta copiar o diretório compilado /app/dist para a imagem final de produção (Multi-Stage Build). No entanto, o diretório não existe naquele momento no estágio anterior porque a build do frontend ou do micro-serviço não foi executada ou falhou.

2 Ações de Correção Passo a Passo (no seu Arquivo Dockerfile)

Ajuste 1: Adicionar script de compilação

Garanta que no primeiro estágio da sua build Docker (*Builder stage*), os comandos necessários para compilação sejam chamados corretamente. Seu arquivo deve conter as seguintes linhas antes de copiar para a imagem final:

# Estágio de Compilação (Builder)
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build # Garante a geração da pasta /app/dist
Ajuste 2: Configuração de Variáveis na Interface do Easypanel

Como seu banco do Easypanel já está sincronizado, verifique se a variável de ambiente DATABASE_URL configurada no Painel do Easypanel está idêntica ao endereço interno do banco de dados (por exemplo, apontando para o contêiner local, ou o IP correto com a porta exposta 5442 como exibido no seu anexo).

Ajuste 3: Direcionamento para Single-Stage (Se aplicável)

Se o seu projeto for puramente um servidor Node TypeScript executável em runtime por tsx / ts-node (onde não é necessário gerar arquivos JavaScript compilados na pasta dist), você pode ajustar o seu Dockerfile para não usar compilação e simplesmente rodar o servidor em modo de desenvolvimento/produção diretamente.
